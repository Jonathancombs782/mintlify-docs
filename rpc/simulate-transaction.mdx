---
title: "simulateTransaction RPC Method"
description: "Simulates a transaction against the current cluster state without broadcasting it, returning logs, errors, account state, and compute units consumed."
---

The `simulateTransaction` method executes a transaction in a sandboxed environment against the current (or specified) bank state without broadcasting anything to the network. Use it to test that a transaction will succeed before paying fees, to estimate compute unit consumption for prioritization fee calculations, or to debug program logic by reading the log output.

## Parameters

<ParamField path="params[0]" type="string" required>
  The transaction to simulate, serialized and encoded. Must match the `encoding` config field (default `base64`).
</ParamField>

<ParamField path="params[1]" type="object">
  Optional configuration object.

  <Expandable title="config fields">
    <ParamField path="sigVerify" type="boolean" default="false">
      When `true`, the node verifies transaction signatures before simulation. Enabling this requires a valid signature and therefore a recent blockhash. When `false` (the default), signature verification is skipped, making it easier to simulate unsigned transactions.
    </ParamField>

    <ParamField path="replaceRecentBlockhash" type="boolean" default="false">
      When `true`, the transaction's `recentBlockhash` is replaced with the node's most recent blockhash before simulation. Useful when you are simulating a transaction that might have an expired blockhash. Cannot be combined with `sigVerify: true`.
    </ParamField>

    <ParamField path="commitment" type="string" default="finalized">
      Commitment level at which to simulate the transaction.
    </ParamField>

    <ParamField path="encoding" type="string" default="base64">
      Encoding of the serialized transaction. Accepted values: `base64` or `base58`.
    </ParamField>

    <ParamField path="accounts" type="object">
      Configuration to return account state after simulation.

      <Expandable title="accounts config fields">
        <ParamField path="encoding" type="string" default="base64">
          Encoding for returned account data. Accepted values: `base64`, `base64+zstd`, `jsonParsed`.
        </ParamField>
        <ParamField path="addresses" type="string[]" required>
          Array of base-58 account addresses to return state for after simulation.
        </ParamField>
      </Expandable>
    </ParamField>

    <ParamField path="minContextSlot" type="number">
      Minimum slot at which to perform the simulation.
    </ParamField>

    <ParamField path="innerInstructions" type="boolean" default="false">
      When `true`, the simulation response includes inner instructions generated by cross-program invocations.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="context" type="object">
  <Expandable title="context fields">
    <ResponseField name="slot" type="number">
      The slot at which the simulation ran.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="value" type="object" required>
  Simulation result.

  <Expandable title="value fields">
    <ResponseField name="err" type="object | null">
      Error if the simulated transaction would fail, or `null` if it would succeed. The shape is the same as a `TransactionError` returned by `getTransaction`.
    </ResponseField>
    <ResponseField name="logs" type="string[] | null">
      Array of log messages produced during simulation, or `null` if log collection was disabled.
    </ResponseField>
    <ResponseField name="accounts" type="array | null">
      Array of account states after simulation, in the same order as the `accounts.addresses` config parameter. Each element is an account object or `null` if the account does not exist. Only present when `accounts` was configured in the request.
    </ResponseField>
    <ResponseField name="unitsConsumed" type="number">
      The number of compute units consumed by the simulated transaction. Use this to set a compute unit limit or calculate a prioritization fee.
    </ResponseField>
    <ResponseField name="returnData" type="object | null">
      Return data set by the last program that called `sol_set_return_data`, or `null`.

      <Expandable title="returnData fields">
        <ResponseField name="programId" type="string">
          Base-58 public key of the program that set the return data.
        </ResponseField>
        <ResponseField name="data" type="array">
          Two-element array `[data, encoding]`.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="innerInstructions" type="array | null">
      Inner instructions generated during simulation. Only present when `innerInstructions: true` was set in the config.
    </ResponseField>
  </Expandable>
</ResponseField>

```javascript JavaScript
const response = await fetch("https://api.devnet.solana.com", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    jsonrpc: "2.0",
    id: 1,
    method: "simulateTransaction",
    params: [
      serializedBase64Transaction,
      {
        encoding: "base64",
        commitment: "confirmed",
        replaceRecentBlockhash: true,
        innerInstructions: true,
        accounts: {
          encoding: "base64",
          addresses: ["83astBRguLMdt2h5U1Tpdq5tjFoJ6noeGwaY3mDLVcri"],
        },
      },
    ],
  }),
});

const { result } = await response.json();

if (result.value.err) {
  console.error("Simulation failed:", result.value.err);
  console.error("Logs:", result.value.logs);
} else {
  console.log("Simulation succeeded");
  console.log("Compute units consumed:", result.value.unitsConsumed);
  console.log("Logs:", result.value.logs);
}
```

<Tip>
Run `simulateTransaction` before [`sendTransaction`](/rpc/send-transaction) during development to catch errors before spending real fees. In production, set `skipPreflight: true` on `sendTransaction` after you have validated the transaction logic, or rely on the built-in preflight simulation that `sendTransaction` runs automatically when `skipPreflight` is `false`.
</Tip>

<Note>
The `unitsConsumed` value from simulation is a reliable estimate for setting a compute unit limit instruction. Add a small buffer (for example, 10–20%) to account for any variation between simulation and actual execution.
</Note>
