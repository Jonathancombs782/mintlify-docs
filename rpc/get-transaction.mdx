---
title: "getTransaction RPC Method"
description: "Returns the full details of a confirmed transaction by its base-58 signature, including instructions, metadata, balances, and log messages."
---

The `getTransaction` method retrieves the complete record of a confirmed transaction. You identify the transaction by its base-58 encoded signature string. The response includes the original transaction message, all pre- and post-execution account balances, log messages, inner instructions, and the transaction error (if any). Use this method to verify that a specific transaction succeeded, to read its output, or to audit historical activity.

## Parameters

<ParamField path="params[0]" type="string" required>
  The transaction signature to look up, as a base-58 encoded string.
</ParamField>

<ParamField path="params[1]" type="object">
  Optional configuration object.

  <Expandable title="config fields">
    <ParamField path="encoding" type="string" default="json">
      Encoding for the returned transaction. Accepted values: `json`, `jsonParsed`, `base58`, `base64`.
    </ParamField>

    <ParamField path="commitment" type="string" default="finalized">
      Commitment level. Only `confirmed` and `finalized` are accepted. `processed` is not supported.
    </ParamField>

    <ParamField path="maxSupportedTransactionVersion" type="number">
      Maximum transaction version to return. Omit to return only legacy transactions. Set to `0` to include version 0 (versioned) transactions. Returns an error if the transaction's version exceeds this value.
    </ParamField>
  </Expandable>
</ParamField>

## Response

Returns `null` if the transaction is not found or is not yet confirmed to the requested commitment level.

<ResponseField name="slot" type="number" required>
  The slot in which this transaction was confirmed.
</ResponseField>

<ResponseField name="blockTime" type="number | null">
  Estimated Unix timestamp (seconds since epoch) of the block that contains this transaction.
</ResponseField>

<ResponseField name="transaction" type="object" required>
  The transaction data. Shape depends on `encoding`. For `json` this is an object with `message` and `signatures` fields.

  <Expandable title="transaction fields">
    <ResponseField name="signatures" type="string[]">
      Array of base-58 encoded signatures. The first signature is the transaction ID.
    </ResponseField>
    <ResponseField name="message" type="object">
      The transaction message containing account keys, instructions, and the recent blockhash.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta" type="object | null" required>
  Transaction status metadata. `null` if metadata is not available.

  <Expandable title="meta fields">
    <ResponseField name="err" type="object | null">
      Error information if the transaction failed, or `null` if it succeeded.
    </ResponseField>
    <ResponseField name="fee" type="number">
      Transaction fee paid in lamports.
    </ResponseField>
    <ResponseField name="preBalances" type="number[]">
      Array of lamport balances for each account in the transaction, before execution.
    </ResponseField>
    <ResponseField name="postBalances" type="number[]">
      Array of lamport balances for each account in the transaction, after execution.
    </ResponseField>
    <ResponseField name="innerInstructions" type="array | null">
      List of inner instructions generated by cross-program invocations. `null` if inner instruction recording was not enabled.

      <Expandable title="inner instruction fields">
        <ResponseField name="index" type="number">
          Index of the outer instruction that generated these inner instructions.
        </ResponseField>
        <ResponseField name="instructions" type="array">
          List of inner instructions with `programIdIndex`, `accounts`, and `data` fields.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="logMessages" type="string[] | null">
      Array of log messages emitted by the transaction's programs. `null` if log recording was not enabled.
    </ResponseField>
    <ResponseField name="preTokenBalances" type="array">
      SPL Token account balances before transaction execution.
    </ResponseField>
    <ResponseField name="postTokenBalances" type="array">
      SPL Token account balances after transaction execution.
    </ResponseField>
    <ResponseField name="rewards" type="array | null">
      Any rewards credited during this transaction. Typically `null`.
    </ResponseField>
    <ResponseField name="loadedAddresses" type="object">
      Addresses loaded from address lookup tables. Contains `writable` and `readonly` arrays of base-58 strings.
    </ResponseField>
    <ResponseField name="returnData" type="object | null">
      The return data set by the last program that set return data, or `null`.

      <Expandable title="returnData fields">
        <ResponseField name="programId" type="string">
          The program that set the return data.
        </ResponseField>
        <ResponseField name="data" type="array">
          Two-element array `[data, encoding]`, e.g. `["AQID", "base64"]`.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="computeUnitsConsumed" type="number">
      Total compute units consumed by this transaction.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="version" type="string | number">
  Transaction version. `"legacy"` for transactions before versioned transactions were introduced, or a number for versioned transactions.
</ResponseField>

## Example

<CodeGroup>

```bash cURL
curl https://api.mainnet-beta.solana.com \
  -X POST \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "getTransaction",
    "params": [
      "5h6xBEauJ3PK6SWCZ1PGjBvj8vDdWG3KpwATGy1ARAXFSDwt8GFXM7W5ZlM7nAqToBnBCZhbFXgOxFnVsBT",
      {
        "encoding": "json",
        "commitment": "finalized",
        "maxSupportedTransactionVersion": 0
      }
    ]
  }'
```

```javascript JavaScript
const response = await fetch("https://api.mainnet-beta.solana.com", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    jsonrpc: "2.0",
    id: 1,
    method: "getTransaction",
    params: [
      "5h6xBEauJ3PK6SWCZ1PGjBvj8vDdWG3KpwATGy1ARAXFSDwt8GFXM7W5ZlM7nAqToBnBCZhbFXgOxFnVsBT",
      {
        encoding: "json",
        commitment: "finalized",
        maxSupportedTransactionVersion: 0,
      },
    ],
  }),
});

const { result } = await response.json();
if (result === null) {
  console.log("Transaction not found or not yet finalized");
} else if (result.meta.err !== null) {
  console.log("Transaction failed:", result.meta.err);
} else {
  console.log("Transaction succeeded in slot", result.slot);
}
```

</CodeGroup>
